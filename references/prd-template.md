## 섹션별 작성 팁

### 경쟁사 분석 (§2)
- 반드시 web_search + web_fetch로 실제 사이트를 방문한다
- 가격 페이지, FAQ, 이용약관까지 확인한다
- 경쟁사 빈틈(gap)이 곧 차별화의 근거다

### 버려야 할 가정 (§5)
- 이 섹션이 없는 PRD는 불완전하다
- 소셜 로그인 ≠ 플랫폼 데이터 접근
- OAuth ≠ 타사 서비스 세션
- 공개 API 유무를 반드시 확인
- "AI가 모든 X를 자동으로" → 범위 제한 필요

### 데이터 모델 (§9)
- SQL CREATE TABLE 형태로 작성 (바로 마이그레이션 가능)
- 각 필드에 주석
- 외래키 관계 명시
- Phase별 테이블 분리 (MVP 테이블 / 이후 확장 테이블)

### 마일스톤 (§14)
- 솔로 개발: 주 15~20시간 기준
- Phase당 2~3주
- 각 Phase 끝에 🎯 마일스톤 목표 (알파/베타/출시 등)
- 체크박스 [ ] 형태로 태스크 나열

### 수익 시뮬레이션 (§15)
- 서버 비용을 반드시 포함 (무료 티어 활용 시 ₩0도 명시)
- 6개월, 1년 시점의 시뮬레이션
- 보수적으로 추정 (전환율 3~7%)
